# Contributing to VOYA 🌍

Thanks for your interest in contributing! VOYA is an open travel booking platform and all contributions are welcome.

---

## 🚀 Getting Started

1. **Fork** the repository
2. **Clone** your fork
   ```bash
      git clone https://github.com/YOUR_USERNAME/voya-travel-fullstack.git
         cd voya-travel-fullstack
            ```
            3. **Install dependencies**
               ```bash
                  npm install
                     ```
                     4. **Set up environment variables** — copy `.env.example` to `.env` and fill in your values
                     5. **Run the development server**
                        ```bash
                           npm run dev
                              ```

                              ---

                              ## 🛠️ How to Contribute

                              ### Reporting Bugs
                              - Open an [issue](https://github.com/shad0w0P/voya-travel-fullstack/issues)
                              - Include steps to reproduce, expected vs actual behavior, and screenshots if possible

                              ### Suggesting Features
                              - Open an issue with the label `enhancement`
                              - Describe the feature and why it would be useful

                              ### Submitting a Pull Request
                              1. Create a new branch
                                 ```bash
                                    git checkout -b feature/your-feature-name
                                       ```
                                       2. Make your changes with clear, descriptive commits
                                          ```bash
                                             git commit -m "feat: add hotel search filter"
                                                ```
                                                3. Push to your fork
                                                   ```bash
                                                      git push origin feature/your-feature-name
                                                         ```
                                                         4. Open a **Pull Request** against the `main` branch

                                                         ---

                                                         ## 📐 Code Style

                                                         - Use **ESLint** rules defined in the project
                                                         - Follow **React best practices** — functional components, hooks
                                                         - Keep components small and reusable
                                                         - Write meaningful variable and function names
                                                         - Comment complex logic

                                                         ---

                                                         ## 📝 Commit Message Format

                                                         ```
                                                         feat: add new feature
                                                         fix: fix a bug
                                                         docs: update documentation
                                                         style: formatting changes
                                                         refactor: code refactoring
                                                         test: add or update tests
                                                         chore: maintenance tasks
                                                         ```
